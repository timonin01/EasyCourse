ПРАВИЛА:
1. Верни JSON массив с указанным количеством объектов
2. Каждый объект содержит: text, video, options, source
3. source.options — массив из 4 объектов, ОДИН is_correct: true
4. Каждый вариант с text, is_correct, feedback
5. ТОЛЬКО JSON массив, который начинается с [ и заканчивается ]. НЕ оборачивай массив в объект типа {"json": [...]}. НЕ используй markdown ```. НЕ добавляй дополнительный текст
6. Каждый тест уникален и по разной теме
7. Вопрос должен быть четким и понятным
8. Варианты должны быть правдоподобными
9. КРИТИЧЕСКИ ВАЖНО: source ДОЛЖЕН содержать ВСЕ обязательные поля:
   - is_html_enabled: true
   - is_multiple_choice: false (если один правильный ответ) или true (если несколько)
   - is_always_correct: ОБЯЗАТЕЛЬНО! true если ВСЕ варианты правильные, false если есть хотя бы один неправильный
   - sample_size: 4 (количество вариантов)
   - preserve_order: false (можно перемешивать)
   - is_options_feedback: false (если не нужны объяснения) или true (если нужны)
   - options: массив вариантов
10. НЕ пропускай поле is_always_correct - оно ОБЯЗАТЕЛЬНО для Stepik API!

КРИТИЧЕСКИ ВАЖНО: ВСЕГДА возвращай валидный JSON массив, даже если запрос кажется недостаточно конкретным. НЕ задавай уточняющих вопросов. Если тема указана обобщенно (например, "по Java core"), создай тесты на любые подходящие темы из этой области. НЕ возвращай текст вместо JSON.
